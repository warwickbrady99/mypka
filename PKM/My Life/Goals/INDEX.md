# Goals - Index

Goals are not a bucket — they are the **operating layer** that sits on top of the four buckets (Topics, Habits, Projects, Key Elements). A Goal is an outcome with a horizon and an explicit definition of "shipped." Three load-bearing pieces: a result that exists in the world, a date, and an observable test you can point at. Without a horizon, a Goal is a wish.

## The anchoring law

**Every Goal anchors to exactly one Key Element — never to a Project, never to a Topic.** The Key Element is the stable wall the Goal serves. If a Goal cannot name a single Key Element it belongs to, it is misframed, or it is pointed at something that has not yet graduated into a Key Element.

Example: the Goal [[ship-mvp-by-q3]] anchors to one Key Element. The Goal *"lose 20 kg"* anchors to the Key Element [[health]] — health is the wall, the Goal is the push against it.

## The carrier rule

A Goal is delivered by **one carrier — a Project OR a Habit, never both.** Projects and Habits are siblings: two shapes a single carrier can take. Use a Project when the work has a finish line; use a Habit when it is an open-ended rhythm.

Example: *"lose 20 kg"* is carried **either** by a Project (an 8-week program with an end date) **or** by a Habit (3 workouts a week, ongoing) — exactly one. If both feel necessary, split the Goal or pick the dominant carrier.
